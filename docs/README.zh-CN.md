# copy-path-line

`copy-path-line` 是一个 VS Code 插件，用来复制“当前文件路径 + 行号”。

它适合在这些场景里使用：

- 在聊天工具里贴代码定位
- 在 GitHub Issue / PR / Review 评论里贴文件引用
- 在团队协作中快速说明某一段代码位置

插件默认复制相对路径，也支持复制绝对路径；支持多光标、多选区，并会把行号整理成紧凑格式，例如 `1-3,5-6,8-11`。

项目地址：

- <https://github.com/dmxiaoshubao/copy-path-line>

## 功能特性

- 支持复制相对路径 + 行号
- 支持复制绝对路径 + 行号
- 支持快捷键触发
- 支持命令面板触发
- 支持编辑器右键菜单触发
- 支持多选区和多光标
- 自动合并重叠或相邻的行号段
- 成功复制时静默，不弹提示，不打断编辑流程

## 命令

插件只提供一套命令 ID，但命令名称会根据 VS Code 语言环境自动切换。

`复制相对路径+行号`
`复制绝对路径+行号`

对应命令 ID：

- `copyPathLine.copyRelativePathWithLine`
- `copyPathLine.copyAbsolutePathWithLine`

## 默认快捷键

默认快捷键绑定到“复制相对路径+行号”：

- macOS: `shift+cmd+c`
- Windows / Linux: `shift+ctrl+c`

“复制绝对路径+行号”命令默认不绑定快捷键。如果你需要，可以在 VS Code 的键位设置里为下面这个命令单独配置：

```text
copyPathLine.copyAbsolutePathWithLine
```

## 右键菜单

插件会在编辑器右键菜单中增加两个入口：

- `复制相对路径+行号`
- `复制绝对路径+行号`

这两个入口和命令面板里的命令行为完全一致。

## 输出格式

输出统一使用以下格式：

```text
路径:行号
```

示例：

```text
src/extension.ts:8
src/extension.ts:1-3
src/extension.ts:1-3,5-6,8-11
/Users/name/project/src/extension.ts:8
```

## 行号规则

### 1. 单光标

如果没有选中文本，只是把光标放在某一行，那么输出当前行号。

例如：

```text
src/app.ts:8
```

### 2. 单个连续选区

如果你选中了连续多行，会输出一个连续区间：

```text
src/app.ts:3-10
```

### 3. 多个离散选区

如果你选中了多个不连续区间，例如：

- `1-3`
- `5-6`
- `8-11`

那么输出会是：

```text
src/app.ts:1-3,5-6,8-11
```

### 4. 自动排序与合并

插件会自动处理这些情况：

- 选区顺序打乱
- 多个选区重叠
- 多个选区正好相邻

最终输出会自动排序，并合并成最简洁的结果。

### 5. 结尾在下一行列号 0 的处理

这是编辑器选区里一个很常见的边界情况。

如果一个选区从第 1 行开始，结束在第 4 行的第 0 列，那么通常意味着你实际选中了第 1 到第 3 行，而不是第 4 行。

插件会按这个编辑器惯例处理，避免多算一行。

## 路径规则

### 相对路径

相对路径模式下：

- 单工作区时，相对路径基于当前 workspace 根目录计算
- 多根工作区时，基于当前文件所属的那个 workspace folder 计算

例如：

```text
src/feature/user/service.ts:12
```

### 绝对路径

绝对路径模式下，直接输出文件在本机上的完整路径：

```text
/Users/name/project/src/feature/user/service.ts:12
```

### 不在工作区中的文件

如果当前文件不属于任何 workspace folder，而你执行的是“复制相对路径+行号”，插件会自动退化为绝对路径输出。

这样做的好处是：

- 不需要弹窗打断
- 不会复制出无意义的相对路径
- 行为稳定、安静

## 使用方式

你可以通过三种方式触发：

### 1. 快捷键

直接按：

- macOS: `shift+cmd+c`
- Windows / Linux: `shift+ctrl+c`

### 2. 命令面板

打开命令面板，搜索：

- `复制相对路径+行号`
- `复制绝对路径+行号`

### 3. 编辑器右键菜单

在代码编辑器里右键，选择：

- `复制相对路径+行号`
- `复制绝对路径+行号`

## 设计原则

这个插件的实现遵循下面几个原则：

### 安静

- 成功时不弹提示
- 不创建多余 UI
- 不打断正常编辑流程

### 轻量

- 只在执行命令时工作
- 不依赖后台轮询
- 不注册不必要的长期监听

### 可测试

核心逻辑拆成纯函数，方便单元测试覆盖：

- 行号区间收集
- 行号区间合并
- 行号格式化
- 相对路径与绝对路径解析

## 适用限制

当前插件只对本地文件编辑器生效，也就是 `file` scheme 的文档。

这意味着下面这类场景不会执行复制：

- 未关联本地文件路径的编辑器
- 非文件类型的虚拟文档

在这些情况下，插件会提示当前没有可复制路径的文件编辑器。
